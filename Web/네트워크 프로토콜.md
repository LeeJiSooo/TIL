## 네트워크 프로토콜의 정의

네트워크 프로토콜은 네트워크 상에서 데이터를 주고받는 규칙과 절차를 정의하는 체계이다.

네트워크 프로토콜은 다양한 통신 장치들이 서로 원활하게 데이터를 송수신할 수 있도록 표준화된 규약을 제공한다.

## TCP/IP 4계층 모델과 관심사의 분리

TCP/IP 모델은 복잡한 네트워크 통신 과정을 4개의 역할(계층)로 쪼개어 관리를 한다. 각 계층이 독립되어 있기 때문에 하위계층(Wi-Fi → 5G)이 바뀌어도 상위계층(크롬 브라우저의 HTTP)의 코드를 수정할 필요가 없는 강력한 유지보수성을 가진다.

#### 4계층 : 애플리케이션 계층(Application Layer)

- 사용자가 직접 맞닿는 영역으로 서비스(웹, 이메일, 파일 전송 등)의 데이터를 생성하고 해석한다. 데이터를 저쪽으로 보내달라고 요청하는 역할
- 대표 프로토콜 : HTTP, HTTPS, SSH, FTP, DNS

#### 3계층 : 전송 계층(Transport Layer)

- 양 끝단의 송수신자 사이에 데이터가 안전하고 정확하게 전달되도록 프로세스 간 통신을 책임진다. 어떤 방식으로 보낼지 규칙을 정한다.
- 대표 프로토콜 : TCP(신뢰성 중시), UDP(속도 중시)

#### 2계층 : 인터넷 계층(Internet Layer)

- 데이터의 최종 목적지 주소(IP)를 확인하고 전 세계 네트워크 중 어떤 길로 가는게 가장 빠를지 경로를 설정(라우팅)한다. → 우편물 분류하고 최적의 항공편, 배편 처리해주는 우체국과 같음
- 대표 프로토콜 : IP, ICMP

#### 1계층 : 네트워크 인터페이스 계층(Network Interface Layer)

- 상위 계층에서 가공된 데이터를 실제 전기 신호나 광 신호로 바꾸어 물리적인 케이블, 랜선, Wi-Fi주파수에 실어 보낸다. → 우체부가 편지 들고 직접 옮기는 단계
- 대표 기술 : Ethernet, Wi-Fi

## TCP vs UDP

#### TCP(Transmission Control Protocol) “꼼꼼하고 신뢰성 있는 전송”

- 연결 지향형 프로토콜. 주고받기 전에 사전에 통로를 개설하는 3-Way Handshake를 거치고 통신이 끝나면 4-Way Handshake로 안전하게 연결을 끊는다.
- 데이터가 중간에 유실되면 다시 재전송을 요청하여 데이터의 완전성(기밀성과 무결성)을 보장한다.
- 웹 페이지 조회(HTTP), 파일 다운로드 , 이메일 등(이미지가 깨지거나 코드가 유실되면 안 되는 서비스)

#### UDP(User Datagram Protocol) “사전 약속 없는 초고속 일방 통행’

- 비연결형 프로토콜. 상대방이 데이터를 받을 준비가 됐는지 확인도 안 하고 잘 받았는지 ACK도 받지 않은 채 일단 데이터를 던진다. 절차가 단순해 속도가 빠르다.
- 지연 시간(Latency)이 최소화된다. 데이터 몇 개가 유실되더라도 끊김 없는 연속성이 중요할 때 사용한다.
- 유튜브 라이브 스트리밍, 온라인 게임, 실시간 영상 통화 등(화면이 아주 잠깐 끊기거나 캐릭터가 순간 이동하더라도 방송/게임이 아예 멈추는 것보다 나은 서비스)

## 애플리케이션 계층의 핵심 프로토콜

#### HTTP(Hypertext Transfer Protocol)

- 웹 브라우저와 웹 서버가 HTML, 이미지, 비디오 등의 데이터를 주고받기 위해 사용하는 표준 프로토콜
- 데이터를 암호화되지 않은 평문 상태로 전송하기 때문에 중간에 누군가 패킷을 가로채면 비밀번호나 개인정보를 그대로 노출당하는 보안 취약점이 있다.

#### HTTPS(HTTP Secure)

- HTTP에 SSL/TLS 암호화 레이어를 추가하여 보안성을 극대화한 프로토콜이다.
- 데이터를 암호화된 보안 봉투에 넣어 전송하므로 기밀성과 무결성이 보장된다. 오늘날 인터넷 표준으로 자리 잡았으며, 대칭키/공개키 매커니즘을 사용해 신원을 검증한다. 사설 주소(HTTP)로 접근해도 HTTPS로 자동 리다이렉트된다.

### SSH(Secure Shell)

- 네트워크 너머에 있는 원격 서버에 안전하게 접속하고 제어하기 위해 사용하는 보안 프로토콜이다.
- AWS EC2 인스턴스 같은 클라우드 서버에 접속할 때 주로 사용한다. 비밀번호나 명령어가 오가는 모든 통신 과정을 암호화하여 해커가 중간에 명령어를 가로채거나 위조하는 것을 차단한다.

## 정리

TCP/IP 4계층은 관심사의 분리를 통해 네트워크 인프라와 애플리케이션 서비스의 독립성을 보장한다. 이때 데어터의 정확성이 중요하다면 전송 계층에서 TCP를 실시간 속도가 중요하다면 UDP를 사용한다. 최종적으로 사용자는 웹 통신의 기본인 HTTP, 보안이 강화된 HTTPS, 혹은 원격 서버 제어를 위한 SSH같은 애플리케이션 프로토콜을 통해 서비스를 이용하게 된다.
