## API 정의

API는 서로 다른 프로그램(애플리케이션)이 데이터를 주고받고 기능을 실행하기 위해 미리 정해놓은 규칙, 약속이다.

#### 웹 API

- 인터넷(네트워크)을 통해 서로 다른 컴퓨터에 있는 프로그램끼리 소통하는 약속이다.
- ex) 기상청 날시 api, 카카오 소셜 로그인 api, 토스 결제 api
- 날씨 데이터를 얻기 위해 위성을 띄우거나 결제 시스템을 위해 카드사와 직접 계약할 필요 없이 남이 잘 만들어둔 거대한 기능을 가져다 내 서비스에 쓸 수 있다.

#### 언어/라이브러리 API

- 소스코드 레벨에서 특정 언어나 라이브러리가 제공하는 기능을 안전하게 쓰기 위한 약속이다.
- Java Stream API의 map() : map() API의 명세는 원본은 건드리지 않고 인자로 받은 규칙에 따라 데이터를 일괄 반환하여 새 결과물을 반환한다.
    - 개발자는 오라클이 내부적으로 메모리를 어떻게 쓰고 반복문을 돌리는지 알 필요 없이 오직 이 api 명세만 믿고 [’사과’, ‘딸기’]를 [’사과 주스’, ‘딸기 주스’]로 안전하게 변환하여 사용한다.
    

## 웹 API요청시 알아야할 요소

1. **Endpoint (주소):** "어디로 요청을 보내야 하는가?" (예: /api/v1/weathers)
2. **Method (행위):** "어떤 작업을 원하는가?" (앞서 배운 GET, POST, PUT, DELETE 등)
3. **Header (메타데이터):** "보내는 데이터의 타입은 무엇이고, 인증 정보는 무엇인가?" (예: Content-Type: application/json, Authorization: Bearer <토큰>)
4. **Body (본문 데이터):** "생성하거나 수정할 구체적인 내용이 무엇인가?" (주로 JSON 형태로 전달, 단 GET이나 DELETE는 보통 Body가 비어있음)

## 정리

- API는 서로 다른 애플리케이션이나 코드 간의 소통을 위해 정의된 인터페이스이자 약속이다. 내부 구현 방식에 의존하지 않고 명세된 규칙에 따라 데이터와 기능을 주고받기 때문에 시스템 간의 결합도를 낮춰준다.
- 외부의 날씨, 결제, 로그인 같은 복잡한 인프라를 API를 통해 쉽게 가져다 쓸 수 있어 개발자가 서비스의 핵심 비즈니스 로직에만 집중할 수 있도록 돕는 핵심 기술이다.
