## HTTP 자격 증명 헤더

클라이언트와 서버가 서로 신분증(인증 정부)을 요청하고 제출할 때 사용하는 HTTP메시지 내의 전용 헤더 공간

HTTP는 무상태성을 가지기 때문에 마이페이지를 조회할 때마다 “나 00 맞다니까?”라는 것을 매번 증명해야 된다. 이때 서로 대화하는 수단이 아래 2가지 핵심 헤더이다.

- WWW-Authenticate(서버 → 클라이언트)  : 서버가 클라이언트에게 “너 인증 필요해. 그리고 어떤 방식으로 신분증을 제출해줘”하고 요청하는 헤더이다.
- Authorization(클라이언트 → 서버) : 클라이언트가 서버에게 “여기 네가 요구한 인증정보가 있어”하고 제출할 때 쓰는 헤더이다.

### Bearer토큰 방식 : Authorization: Bearer<JWT토큰>

매번 id/pw를 보내는 위험을 감수하는 대신 최초 로그인시 서버로부터 토큰이라는 단 한장의 안전한 디지털 열쇠를 발급받아 그 후 모든 요청 헤더에 이 토큰을 실어 보내는 방식이다.

### OAuth(소셜 로그인 프로토콜) : 카카오, 구글, 네이버 소셜 로그인의 기반이 되는 보안 프로토콜이다.

단순이 인증을 넘어 “내가 00서비스 비번을 직접 내 서비스에 알려주지는 않겠지만 00서비스 이메일이랑 프로필 사진을 가져가서 쓸 수 있게 인가해줄게”하는 방식이다. 이 OAuth 프로토콜의 최종 결과물로도 결국 내부적으로는 Bearer토큰을 발급받아 통신하게 된다.
