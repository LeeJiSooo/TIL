## INTERNET정의

전세계 컴퓨터 네트워크들이 상호 연결되어 데이터를 주고받을 수 있게 해주는 거대한 네트워크 시스템

TCP/IP라는 표준 프로토콜을 사용하여 다양한 컴퓨터 네트워크들이 서로 통신할 수 있게 한다.

## 사용 이유

정보 검색, 소통, 온라인 거래 등 다양한 정보에 접근하기 위해서이다.

## 동작 방식

1. 인트라넷 내부 요청 : 사용자가 내부 네트워크(인트라넷)에서 인터넷에 접속하려고 할 때 요청은 내부 네트워크 장비를 먼저 거친다.
    
    google.com을 입력하면 컴퓨터는 google.com이 어디있는거지?라는 질문을 던진다.
    
    → 우리의 컴퓨터가 먼저 내부 네트워크나 통신가(ISP)가 지정한 DNS서버가 있고 이 서버에 google.com의 ip주소가 무엇인지?를 물어본다. → DNS서버는 이 주소를 찾아서 우리의 컴퓨터에게 알려준다.
    
2. 게이트웨이를 통한 이동 : 요청이 인트라넷, 인터넷을 연결하는 게이트웨이를 통과한다. 게이트웨이는 인터넷에서 나가는 트래픽을 라우팅하고 보안검사나 프로토콜 변환을 수행한다.
    
    → 우리의 컴퓨터는 특정한 내부 네트워크에 포함되어 있기 때문에 마음대로 외부 인터넷 세상으로 나갈 수 없어서 게이트웨이(문)을 거쳐야 한다. → 회사의 라우터나 방화벽이 게이트웨이 역할을 한다. 게이트웨이는 “내부 직원이 구글에 접속하려고 하네? 허용된 요청이니 보내주자”고 판단해서 요청 데이터를 외부로 보낸다. 이 과정에서 보안검사를 할 수도 있고 내부에서 쓰는 사설 IP주소를 외부 인터넷에 통용되는 공인 IP주소로 바꿔준다(NAT)
    
3. 외부 네트워크 접속 : 게이트웨이를 거친 요청은 다양한 외부 네트워크와 라우터를 거쳐 최종적으로 목적지 웹 서버에 도달한다.
    
    → 이 요청이 한번에 전달되는게 아니라 패킷이라는 단위로 나눠서 전송된다. → 이 패킷들이 전 세계에 깔린 수많은 라우터를 거쳐서 목적지로 간다. → 라우터는 목적지 주소를 보고 가장 효율적인 다음 경로로 패킷을 전달해주는 역할을 한다.
    
4. 응답의 반환 : 웹 서버는 요청된 웹 페이지나 데이터를 사용자에게 되돌려 보내며 이 응답 역시 게이트웨이를 통해 사용자의 인트라넷으로 돌아온다.

트래픽 ? 서버의 데이터 전송량

라우터 ? 컴퓨터 네트워크 간에 데이터 패킷을 전송하는 네트워크 장치

## 인터넷 vs 웹

인터넷 : 전 세계 컴퓨터 네트워크들이 서로 연결된 하드웨어와 소프트웨어의 집합체로 웹을 포함한 다양한 네트워크 서비스와 응용프로그램을 지원하는 기반 구조이다.

웹 : 인터넷을 통해 정보를 공유하는 하나의 서비스로 웹 페이지들이 하이퍼링크로 연결되어 있는 시스템이며 이 정보는 주로 HTML문서의 형태로 존재한다.

## 인터넷 vs 인트라넷

인터넷  : 전 세계의 컴퓨터 네트워크들이 서로 연결되어 정보를 교환할 수 있는 공개적이고 글로벌한 네트워크이다.

인트라넷 : 특정 조직이나 기업 내부에서만 사용되는 사설 네트워크이다. 조직의 구성원들이 정보를 공유하고 협업을 용이하게 하는 내부 커뮤니케이션 도구로 활용된다.

## Gateway

서로 다른 네트워크 또는 시스템 간의 연결점 또는 진입점 역할을 하는 물리적인 장치나 소프트웨어를 의미한다.

### API Gateway

수백개의 마이크로 서비스로 쪼개진 복잡한 시스템이 있으면 이 시스템의 단일 진입점 역할을 한다. 클라이언트의 모든 요청을 API게이트웨이가 받으면 다음 요청에 맞는 내부 서비스로 안전하게 연결해준다.
