## 타입

타입은 변수나 값이 가질 수 있는 데이터의 종류와 형식을 지정하는 개념이다.

자바는 정적 타입으로, 모든 변수는 사용 되기 전에 타입을 명확하게 선언해야한다.

타입은 데이터의 해석방식, 연산 가능 여부, 메모리 방식을 결정하며, 컴파일 시점에 타입 오류를 잡아 코드의 안정성과 예측 가능성을 높여준다

### 기본 타입Primitive Type)

기본 타입은 값 그 자체를 저장하는 타입으로, 자바에서 가장 단순하다.

기본 타입은 변수 선언 시 값을 직접 저장하며, 객체가 아니므로 메서드를 호출할 수 없다.

| 타입 | 설명 | 예시 |
| --- | --- | --- |
| `int` | 정수 (4바이트) | `int age = 25;` |
| `double` | 실수 (8바이트) | `double pi = 3.14;` |
| `char` | 문자 (2바이트, 유니코드) | `char grade = 'A';` |
| `boolean` | 참/거짓 (1비트 개념) | `boolean isOpen = true;` |
| `byte` | 작은 정수 (-128~127) | `byte flag = 1;` |
| `short` | 정수 (2바이트) | `short count = 1000;` |
| `long` | 큰 정수 (8바이트) | `long total = 100000000L;` |
| `float` | 실수 (4바이트) | `float rate = 0.75f;` |

### 참조 타입(Reference Type)

참조 타입은 값 자체가 아닌, 값이 저장된 메모리 주소(참조)를 저장하는 타입이다.

| 타입 | 설명 | 예시 |
| --- | --- | --- |
| 클래스 | 사용자가 정의하거나 자바에서 제공하는 설계도로, 객체를 생성하는 기반이 되는 타입 | `String name = "Alice";Scanner sc = new Scanner(System.in);` |
| 배열 | 동일한 타입의 값을 연속적으로 저장하며, 인덱스로 접근할 수 있는 자료 구조 | `int[] scores = {90, 80, 70};String[] names = {"Tom", "Jane"};` |
| 인터페이스 | 구현 클래스가 따라야 할 규칙(메서드 시그니처 등)을 정의하는 타입 | `Runnable task = new MyTask();List<String> list = new ArrayList<>();` |
| 열거형 (enum) | 미리 정의된 상수 집합을 갖는 타입으로, 선택 가능한 값이 제한되어 있음 | `Day today = Day.MONDAY;enum Day { MONDAY, TUESDAY, ... }` |

참조 타입은 메서드 호출, 속성 접근이 가능하다.

## 정보1 : final

final 키워드는 한 번 초기화한 변수나 필드의 값을 이후에 변경할 수 없도록 제한하는 자바의 제한자이다. → 값을 단 한 번만 할당할 수 있으며, 이후에는 다시 대입할 수 없다.

```jsx
final int MAX_SPEED = 120;
MAX_SPEED = 150 // 오류 발생 : final 변수는 값 변경 불가
```

final이 붙은 참조 타입 변수의 경우, 참조값을 변경할 수 없지만, 객체 내부 상태는 변경 가능하다.

```jsx
final StringBuilder sb = new StringBuilder("Hi");
sb.append("there") // 내부 내용 변경은 가능
sb = new StringBuilder("Hello"); // 오류 : 참조 자체를 변경할 수는 없음
```

## 정보2 : static

static 키워드는 클래스 단위로 공유되는 필드나 메서드를 정의할 때 사용 가능

일반적으로 객체를 생성해야 필드나 메서드에 접근할 수 있지만, static으로 선언된 요소는 객체 없이도 클래스 이름만으로 접근이 가능

```jsx
public class Car{
		static int getMaxSpeed(){
				return 240;
		}
}

int maxSpeed = Car.getMaxSpeed(); // 객체 없이 호출 가능
```

static으로 선언된 필드나 메서드는 JVM이 클래스를 로딩할 때 메서드 영역에 단 한번만 생성함

이후 프로그램이 종료될 때까지 해당 클래스의 모든 객체가 공유하며 사용할 수 있음

## 정보3 : Wrapper

Wrapper 클래스는 int, double, boolean과 같은 기본 자료형을 객체로 감싸는 클래스를 의미한다.

기본 타입은 값 그 자체를 정의하는 반면, 객체로써의 기능(메서드 호출, 제네릭 사용, null 할당 등)을 제공하지 않는다.

ex) ArrayList에는 int를 직접 저장할 수 없고, 대신 Integer를 사용해야한다.

```jsx
ArrayList<Integer> list = new ArrayList<>();
list.add(10); // 기본 10이 Integer 객체로 자동 변환되어 저장됨
```

## 정보4 : 오토박싱, 언박싱

### 오토박싱

오토박싱은  기본 타입을 자동으로 래퍼 클래스 객체로 변환하는 기능이다.

int 값을 Integer객체에 자동으로 감싸주는 동작을 의마한다.

```jsx
Integer num = 10; // int -> Integer(자동 박싱)
```

### 언박싱

언박싱은 래퍼 클래스 객체를 다시 기본 타입으로 자동 변환하는 기능이다.

Integer 객체에서 int값만 꺼내는 동작을 의미한다.

```jsx
int value = num; // Integer -> int(자동 언박싱)
```

사용이유? 

자바는자바는 기본 타입과 참조 타입을 명확히 구분하지만, 컬렉션은 객체(참조타입)만 다룰 수 있다.

즉, int는 List<int>처럼 직접 사용할 수 없기 때문에 List<Integer>로 사용해야 하며, 이때 오토박싱과 언박싱이 자연스럽게 기본 타입과 객체 타입을 연결해주는 역할을 한다.

## 질문

### 프로그래밍 언어에서 타입을 지정해서 사용하는 것의 이점은 무엇인가?

컴파일 단계에서 타입 불일치로 인한 오류를 미리 잡아내어 안정성이 높아지고 변수가 어떤 종류의 데이터를 담고 있는지 명확히 알 수 있어 코드를 이해하고 수정하기 쉽다.

### int와 Integer는 어떤 차이가 있고 각각 어떤 상황에서 더 적절하게 사용할 수 있는가?

- int는 기본타입이며 스택 메모리에 저장되고 null이 불가능하지만 Integer는 참조타입/객체이며 힙 메모리에 주소가 저장되고 null이 가능하다.
- int는 단순한 연산처럼 성능과 메모리 효율이 중요한 일반적인 계산 상황에 적절하고 Integer는 컬렉션 프레임워크에 데이터를 담을 때 적절하다.

### final StringBuilder sb = new StringBuilder(”Hello”); 에서 sb.append는 가능한데 sb = new StringBuilder(”hi”)는 왜 불가능한가?

sb.append **가 가능한 이유:** `final`은 `sb`라는 변수가 가리키고 있는 **힙 메모리 속 StringBuilder 객체의 주소**를 바꾸지 못하게 막는 것이다. `append`는 그 주소로 찾아가서 객체 내부의 문자열 데이터만 변경하는 것이므로 아무런 문제가 없다.

sb = new StringBuilder(”hi”) **가 불가능한 이유:** `new` 연산자를 사용하면 힙 메모리에 완전히 새로운 StringBuilder 객체가 만들어지고, 그 새로운 주소값을 `sb`에 대입하려고 시도하게 된다. `final`로 인해 `sb`가 가진 주소값은 절대 변경할 수 없으므로 컴파일 에러가 발생한다.
