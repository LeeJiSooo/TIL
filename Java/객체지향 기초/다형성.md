## 다형성의 정의

다형성은 하나의 타입으로 여러 객체를 다룰 수 있도록 하여, 동일한 코드가 상황에 따라 다른 동작을 수행하게 하는 객체지향의 특성이다.

자바에서는 주로 부모 클래스나 인터페이스 타입으로 객체를 참조하는 방식으로 다형성이 구현된다.

```jsx
// Car.java
public interface Car {
    void drive(); // 공통 인터페이스: 여러 구현체가 같은 메서드를 다른 방식으로 구현
}

// ElectricCar.java
public class ElectricCar implements Car {
    public void drive() {
        System.out.println("전기차가 조용하게 달립니다.");
    }
}

// GasolineCar.java
public class GasolineCar implements Car {
    public void drive() {
        System.out.println("가솔린차가 힘차게 달립니다.");
    }
}
```

Car라는 인터페이스를 구현한 ElectricCar, GasolineCar같은 여러 클래스가 있다면 Car 타입 하나로 다양한 객체를 참조하고, 동일한 메서드 호출로 서로 다른 동작을 실행할 수 있다.

```jsx
// Main.java
public class Main {
    public static void main(String[] args) {
        Car car; // 상위 타입(Car)으로 선언 → 다형성의 기반

        car = new ElectricCar(); // Car 타입 변수에 ElectricCar 인스턴스를 대입
        car.drive(); // ElectricCar의 drive()가 실행됨 → 동적 바인딩에 따른 다형성

        car = new GasolineCar(); // 같은 Car 타입 변수에 다른 구현체 대입
        car.drive(); // GasolineCar의 drive()가 실행됨 → 동일한 코드, 다른 동작
    }
}
```

Car라는 하나의 타입을 기준으로 다양한 객체를 바꿔 끼울 수 있고, drive()메서드 호출은 각 객체의 구현에 따라 다르게 동작한다.

다형성은 정적 다형성(오버로딩)과 동적 다형성(오버라이딩)으로 나뉘며, 컴파일 타임과 런타임에서 각각 다르게 동작할 수 있다.

<aside>
💡

오버로딩은 메서드 이름은 같지만 매개변수의 형태나 개수를 다르게 정의하여 여러 버전을 만드는 것이다.

오버라이딩은 상속받은 메서드를 자식 클래스에서 다시 정의해 새로운 동작으로 바꾸는 것이다.

</aside>

<aside>
💡

컴파일 타임은 소스 코드를 자바 컴파일러가 바이트 코드로 변환하는 시점을 말한다. 이 단계에서는 문법 오류, 타입 오류 등을 컴파일러가 미리 검사한다. 즉 코드를 실행하기 전, 개발 도구가 코드를 분석하고 오류를 잡는 과정이다.

런타임은 컴파일된 프로그램이 실제로 실행되는 시점을 말한다.

이 단계에서는 메모리 할당, 메서드 호출, 예외 발생 등이 일어난다.

컴파일 시점에는 잡히지 않은 오류가 런타임에서 발생할 수 있다.

</aside>

## 사용 이유

다형성을 사용하는 이유는 코드 수정을 최소화하면서 다양한 기능을 유연하게 확장할 수 있기 때문이다.

일반적으로 구현체마다 조건문을 사용해 분기 처리하면, 새로운 기능이 추가될 때마다 기존 코드를 수정해야 하며 코드가 복잡해지고 변경에 취약한 구조가 된다.

하지만 다형성을 적용하면, 상위 타입을 기준으로 코드를 작성한 뒤 구체적인 구현 객체만 교체하면 되기 때문에 코드를 수정하지 않고도 새로운 기능을 추가하거나 기존 기능을 대체할 수 있다.

## 질문

### Car car = new ElectricCar();에서 컴파일러가 보는 타입과 런타임에 실제로 실행되는 객체 타입이 무엇인가?

- **컴파일러가 보는 타입 (선언 타입):** **`Car`**
    - 컴파일러는 코드의 안전성을 검사하기 때문에 변수의 왼쪽 타입(`Car`)만 봅니다. 따라서 `car` 변수로는 `Car` 클래스에 정의된 메서드만 호출할 수 있다.
- **런타임에 실제로 실행되는 객체 타입 (실제 타입):** **`ElectricCar`**
    - 프로그램이 실행(`java` 명령어 실행)되면 메모리(Heap)에 실제로 생성된 `ElectricCar` 인스턴스를 참조하게 된다.

### 오버로딩 vs 오버라이딩: 런타임 다형성과 연결되는 것은?

- **오버라이딩**은 부모 타입의 메서드를 자식 클래스에서 재정의하는 것이다. 컴파일러는 부모의 메서드로 인식하지만, 실제 프로그램이 실행될 때(Runtime) JVM이 **실제 생성된 자식 객체의 메서드를 찾아서 호출한다.** 이를 다이내믹 바인딩(Dynamic Binding)이라고 하며, 이것이 다형성의 핵심 메커니즘이다.
- 반면, 오버로딩(Overloading)은 메서드 이름은 같고 매개변수만 다르게 하는 것으로, 컴파일러가 소스 코드를 컴파일하는 시점에 어떤 메서드를 호출할지 이미 결정(Static Binding)해 버리기 때문에 런타임 다형성과는 거리가 멀다.
