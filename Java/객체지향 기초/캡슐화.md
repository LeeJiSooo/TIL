## 캡슐화 정의

캡슐화는 객체의 내부 상태를 외부에서 직접 접근하지 못하도록 감추고, 메서드를 통해 간접적으로 접근하도록하는 것이다.

클래스 내부 필드를 private으로 선언하여 외부에서 직접 접근하지 못하도록 막고, 필요한 경우 public 메서드를 통해 제한적으로 접근하거나 수정한다.

## 사용 이유

객체의 내부 상태를 외부로부터 보호하고 데이터의 접근이나 변경을 방지하기 위해서이다.

예를 들어, 자동차 클래스에서 연료랑 같은 필드를 외부에서 마음대로 바꿀 수 있으면 음수 값을 넣는 등 프로그램의 안정성이 떨어질 수 있다.

이때 해당 필드를 private으로 감추고 getFuel()이나 setFuel(int value)처럼 검사 로직이 포함된 메서드로만 접근할 수 있도록 제한하면 잘못된 사용을 방지하면서 정보를 안전하게 전달할 수 있다.

## 사용방법

클래스의 필드(멤버 변수)를 private으로 선언하고 외부에서 직접 접근하지 못하도록 차단한다.

필드에 접근하거나 값을 설정할 수 있도록 public 접근 제어자를 가진 메서드를 정의한다.

```jsx
public class Car {
    // 필드를 private으로 감춤
    private int speed;

    // 외부에서 읽을 수 있도록 getter 제공
    public int getSpeed() {
        return speed;
    }

    // 외부에서 값을 설정할 수 있도록 setter 제공
    public void setSpeed(int speed) {
        if (speed >= 0) {
            this.speed = speed;
        }
    }
}
```

외부에서는 Car 클래스의 speed필드에 직접 접근할 수 없고, getSpeed()나 setSpeed()메서드를 통해 간접적으로 접근할 수 있다.

다만 모든 필드를 setter로 열면 좋지 않고 객체가 제공해야 하는 의미 있는 행위 메소드를 고민해봐야된다.

예를들어 상품의 가격을 setPrice()를 통해 설정한다면 가격이 상승했는지 아니면 할인되어서 줄어든건지 의미를 알 수 가 없다…
