## Interface정의

인터페이스는 구현 세부사항 없이 클래스가 구현해야할 메서드의 집합만을 정의한 설계 명세서이다.

인터페이스는 메서드의 이름과 매개변수만 정의하고 실제 동작은 포함하지 않는다.

오직 해당 인터페이스를 따르는 클래스에서 모든 메서드를 반드시 오버라이딩해서 구현해야 하고 명세서와 구현을 분리함으로써 유연하고 확장 가능한 설계를 가능하게 한다.

## 사용 이유

기능이 완성되지 않은 설계 단계에서도 구조를 먼저 정의하고, 구현자 간 역할을 분리하여 유연하고 확정 가능한 개발을 하기 위해서이다.

인터페이스를 사용하면 명세 중심의 설계가 가능하고, 구현자 간의 작업 분리가 명확해지며 변경에 강한 코드 구조를 만들 수 있다.

## 사용 방법

interface로 선언하고, 클래스에서 implements로 구현한 뒤, 구현한 객체를 인터페이스 타입으로 사용한다.

인터페이스 안에는 본문이 없는 메서드 선언만 존재하며, 이 메서드들은 public abstract이다.

<aside>
💡 public abstract는 어디서든 접근가능한 구현 없이 선언만 된 메서드를 뜻한다. 해당 메서드는 반드시 하위 클래스에서 구현되어야 한다.

</aside>

### 기본 형식

```jsx
public interface Car {
    void startEngine();  // 시동 걸기
    void stopEngine();   // 시동 끄기
}
```

Car가 반드시 구현해야 하는 기능으로 시동걸기와 시동 끄기가 있다면 다음과 같이 인터페이스를 정의할 수 있다.

### 클래스에서 인터페이스 구현

인터페이스를 구현하려면 implements키워드를 사용해야 한다.

인터페이스의 모든 메서드를 오버라이딩해야하며, 각 기능의 실제 동작을 클래스 내부에서 작성한다.

```jsx
public class ElectricCar implements Car {
    @Override
    public void startEngine() {
        System.out.println("배터리를 사용해 시동을 겁니다.");
    }

    @Override
    public void stopEngine() {
        System.out.println("전원을 차단해 시동을 끕니다.");
    }
}
```

### 인터페이스 타입으로 객체 사용

인터페이스는 타입으로도 사용가능하다.

```jsx
Car car = new ElectricCar();
car.startEngine();  // 출력: 배터리를 사용해 시동을 겁니다.
car.stopEngine();   // 출력: 전원을 차단해 시동을 끕니다.
```

상위 개념인 Car타입만 알고 있으면 구현체에 상관없이 일관된 방식으로 다룰 수 있다.

### 여러 인터페이스 구현하기

자바는 클래스는 하나만 상속할 수 있지만 인터페이스는 여러 개를 동시에 구현할 수 있다.

```jsx
public interface Navigation {
    void turnOnNavigation();
}

public class SmartCar implements Car, Navigation {
    @Override
    public void startEngine() {
        System.out.println("스마트카 시동 시작");
    }

    @Override
    public void stopEngine() {
        System.out.println("스마트카 시동 종료");
    }

    @Override
    public void turnOnNavigation() {
        System.out.println("내비게이션을 실행합니다.");
    }
}
```

## 인터페이스 vs 추상클래스

자바에서는 클래스 간의 공통 구조를 정의할 때 인터페이스와 추상클래스를 사용할 수 있다.

인터페이스는 공통된 메서드의 이름만 정의하고 구현체는 강제하지 않는다.

인터페스를 구현한 클래스는 그 인터페이스를 따르는 한 반드시 메서드를 재정의해야한다.

반면 추상클래스는 공통된 메서드 정의 뿐 아니라, 일부 구현도 함께 제공할 수 있다. → 일부 기능은 미리 구현해두고 나머지는 자식 클래스에서 선택적으로 재정의할 수 있다.
