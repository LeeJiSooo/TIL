## Thread정의

스레드는 하나의 프로그램 내에서 동시에 실행할 수 있는 독립적인 실행 흐름 단위이다.

비동기 처리를 가능하게 만드는 기본적인 실행 흐름

비동기처리 : 어떤 작업이 끝날때까지 전체 흐름을 멈추지 않고 그 작업은 따로 진행하게 만들고 현재 흐름은 알아서 다음 일을 하게 만드는 방식이다. → 별도의 흐름을 담당하는 대표적 자원이 스레드이다. 

실무에서 직접 new Thread 하는 일은 거의 없으며 ExecutorService, Thread Pool 정도 사용할 수 있다.

실제로는 Async 어노테이션을 쓰거나 카프카 같은 도구를 사용한다. → 이러한 도구들도 내부적으로는 쓰레드를 사용하고 어떤 실행흐름에서 작업을 돌릴지 큐에 넣을지 몇개까지 쓰레드를 사용할지 예외를 어떻게 처리할지 다룬다.

## 사용 이유

여러 작업을 동시에 실행하여 어플리케이션의 성능을 높이기 위해서 사용한다.

사용자 요청 처리 → 동시에 이메일 발송, 파일 저장, DB에 로그 기록 등 다수의 작업이 동시에 이루어지는 상황이 많다.

이러한 여러 작업을 순차적으로 처리하면 응답이 느려지고 사용자 경험이 떨어지면서 시스템 자원도 비효율적이게 된다.

각 작업을 별도의 스레드로 분리하면 서로 영향을 주지 않으면서 동시에 처리할 수 있다.

### 실무에서 스레드 사용 이유

- 백그라운드 작업 분리 → 사용자 요청과 동시에 실행되는 이메일 발송, 알림 저장, 파일 업로드 같은 작업을 메인 흐름에서 분리하면 응답 속도를 유지할 수 있다.
- 대용량 데이터 병렬 처리 → 배치 작업이나 파일 처리, db연산 같은 무거운 작업을 스레드를 통해 병렬로 분산하면 성능이 크게 향상된다.
- 사용자 경험 개선 → 채팅, 다운로드, 실시간 알림과 같은 ui와 상호작용하는 기능들이 동시에 작동하도록 하려면 스레드 기반 비동기 처리가 필요하다.

## 사용 방법

자바에서는 스레드를 만들기 위해 Thread클래스를 상속하거나, Runnable 인터페이스를 구현한 객체를 Thread에 전달하여 실행할 수 있다.

스레드를 실행하면 기존 흐름과는 독립적으로 병렬처리된다.

### Thread클래스 상속하기

```jsx
class DrivingCar extends Thread {
    @Override
    public void run() {
        System.out.println("자동차 주행 스레드 실행 중: " + Thread.currentThread().getName());
    }
}

public class Main {
    public static void main(String[] args) {
        DrivingCar car = new DrivingCar();
        car.start(); // 새로운 스레드에서 자동차 주행 시작
    }
}
```

car.run()을 호출해도 별도 스레드가 생성되지 않고 현재 스레드에서 실행된다. → 멀티스레드가 되지 않는다.

car.start()를 호출하면 새로운 실행 흐름이 만들어지고 내부적으로 run()메서드가 실행된다 → 멀티스레드가 되지 않는다.

## 공유 변수와 메모리 가시성 문제

동시성은 여러 작업이 겹쳐서 진행되는 상황을 의미한다. 

실제로 같은 순간에 여러 cpu코어에서 병렬로 실행될 수도 있고 하나의 cpu가 아주 빠르게 여러 작업을 번갈아서 처리해서 동시에 실행되는 것처럼 보일 수 있다.

→ 여러 실행 흐름이 같은 시간대에 존재한다.

여러 실행 흐름이 각자 독립적인 데이터만 다룰때는 안전하지만 같은 데이터를 함께 건드릴때는 문제가 생긴다. → 여러 스레드가 동시에 접근하는 변수, 객체, 파일 같은 것들을 공유 자원이라고 부른다.

동기화는 이런 문제를 막기 위해서 여러 스레드가 공유 자원에 접근하는 순서를 조절하는 방법이다.

## 정리

- 스레드는 하나의 프로그램 안에서 동시에 실행될 수 있는 독립적인 실행단위이다.
- 여러 작업을 동시에 처리하여 성능과 응답성을 높이기 위해 사용된다.
- 스레드 내부에서는 예외처리를 별도로 하는것이 필요하다.
- 여러 실행 흐름이 동시에 같은 데이터(공유변수)를 만지면 문제가 생길 수 있다.
- 문제 해결을 위해 동기화, atomatic, volatile, lock 같은 도구를 배우게된다.
